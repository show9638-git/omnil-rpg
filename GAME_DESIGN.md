# 全零〈オムニル〉— 星なき始まり —
## ゲームデザイン・実装設計 v0.1

## 1. 作品の核

**形式**：ドット絵／軽量コマンドバトルRPG

**プレイヤー体験**：
- 世界は広く見えるが、移動操作を要求しすぎない
- 三人が素材・依頼・戦闘・会話を経て「自分の意思」を獲得していく
- 成長は強さだけでなく、感情と人格の変化としても見える

**ゲームの一文要約**：
> 命令に従うだけだった白・黒・虹の三人が、世界を歩きながら“選ぶ力”を手に入れる探索・育成コマンドRPG。

---

## 2. 画面とゲームループ

```text
世界地図
  ├ 宿場町リンドホルム
  │    ├ ギルド：依頼受注／報告／ランク認定
  │    ├ 道具屋：消耗品購入
  │    ├ 製造所：素材から道具・固有装備を製造
  │    ├ 宿屋：HP／MP回復
  │    └ 買取所：素材売却
  ├ 風渡る草原：探索
  ├ 囁きの森：探索
  └ 落星の遺構：探索（E級から）

探索
  ├ 敵遭遇 → コマンド戦闘 → 経験値・素材・G
  ├ 素材発見
  └ 所持金発見

成長
  ├ キャラ経験値 → レベル・パネルポイント
  ├ パネル解放 → ステータス・技・物語能力
  └ 依頼達成 → 冒険者経験値 → ランク・地域・店の解放
```

**二重成長**を採用する。

| 成長軸 | 得られるもの | 意味 |
|---|---|---|
| キャラクターレベル | 基礎能力、パネルポイント、技 | 三人自身の戦闘力と自己形成 |
| 冒険者ランク | 店・製造・依頼・地域の解放 | 世界から得る信用と行動範囲 |

## 3. 三人の戦闘役割

| キャラ | 基本役割 | 種族特性 | 初期技能 | 物語パネル |
|---|---|---|---|---|
| 虹全〈コウゼン〉 | 攻防・連携・調律 | HP半分以下で与ダメージ／回復量上昇 | 調律斬 | 「選択」→ 共鳴波 |
| 白零〈ハクレイ〉 | 守護・回復 | 戦闘後、最も傷ついた味方を回復 | 白光の加護 | 「守りたい」→ 白光の護り |
| 黒零〈コクレイ〉 | 攻撃・侵食・弱体 | 弱体状態の敵へのダメージ上昇 | 漆黒刃 | 「壊したくない」→ 終末の楔 |

### 感情とパネルの結び付け

単にレベルだけを条件にしない。

- 虹全：初勝利で「命令を待たず、自分で進む」ことを選ぶ
- 白零：治療師へ薬草を届ける依頼で「守りたい」を理解する
- 黒零：獣王討伐後、壊れた森を前にして「壊したくない」を理解する

これにより、成長盤がストーリーの進行記録になる。

## 4. 第一章の進行設計

### 開始地点

**宿場町リンドホルム**

三人は情報・補給・行動範囲の確保を目的として到着する。最初は任務遂行の言葉しか使わない。

### F級帯

| 依頼 | 内容 | 報酬 | 物語上の意味 |
|---|---|---|---|
| 薬草を届けて | 草原の薬草を3つ納品 | 80G・冒険者経験30・回復薬 | 白零が「助ける」ことを知る |
| 風を裂く牙 | 風斬りウルフを3体討伐 | 130G・冒険者経験60・エーテル | 三人が町から信頼される |

合計90経験値で **E級** へ昇格。

### E級帯

| 解放物 | 内容 |
|---|---|
| 落星の遺構 | 古びた石片・虹晶の粉が採取可能な高危険地帯 |
| 落星の欠片 | 遺構の石片を2つ集める依頼 |
| 森の獣王 | 囁きの森の苔牙の獣王を討伐する依頼 |

獣王戦の後、黒零の物語パネルが解放条件を満たす。

## 5. 実装済みデータ構造

`js/data.js` にゲームデータを集約する。

| データ | 主な用途 |
|---|---|
| `CHARACTER_DEFS` | 基礎能力、成長率、種族特性、パネル |
| `SKILLS` | MP、対象、威力、回復量、弱体効果 |
| `ENEMIES` | 能力、経験値、ドロップ、行動 |
| `LOCATIONS` | マップ座標、ランク条件、探索テーブル |
| `QUESTS` | 依頼条件、報酬、開放条件 |
| `RECIPES` | 素材、製造物、装備効果 |

地域・敵・依頼・装備を増やす時は、原則としてこのデータファイルを追加編集する。

## 6. v0.1で意図的に抑えたもの

この段階では「何を遊ぶゲームか」を先に成立させるため、以下は次段階に回す。

- 歩行アニメーションと自由移動
- 大量の敵アニメーション
- 複雑な連携技
- 装備の付け替え画面
- 音楽・SE
- ロード／セーブファイルの書き出し
- 正式立ち絵・イベントCG

重いアクション処理を使わず、スマホブラウザで動きやすい構成を維持する。
